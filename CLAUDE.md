# 자동차 부품 표준류 자동생성 시스템 — 프로젝트 기획서> 이 파일은 Claude Code가 프로젝트 전체 맥락을 이해하기 위한 기준 문서입니다.> 작업 시작 전 반드시 이 파일을 먼저 읽으세요.\---## 프로젝트 개요### 목표자동차 부품 제조사의 품질 표준류(FMEA, Control Plan, 작업표준서, 자주검사항목)를도면(PDF)과 공정검토서(Excel)를 입력받아 챗봇 대화로 자동 생성하는 SaaS 시스템 구축### 판매 대상국내 자동차 부품 제조사 (1차, 2차 협력사)### 핵심 가치* 표준류 작성 시간 80% 단축* 숙련 엔지니어 경험을 AI로 전수* 고객사별 요구사항(CSR) 자동 반영\---## 입력 / 출력### 입력 (Input)* 도면 : PDF 파일 (치수, 공차, 재질, 특기사항 포함)* 공정검토서 : Excel 파일 (공정순서, 설비, 공정특성 포함)* 공장 프로파일 : 공장 유형, 주요 설비, 고객사 정보> ⚠️ 공정검토서는 표준류가 아님. 입력 소스(기준 문서)임.> 표준류 = FMEA, CP, 작업표준서, 자주검사항목 4종만 해당.### 출력 (Output)1. FMEA (고장유형 및 영향분석) — Excel2. Control Plan (관리계획서) — Excel3. 작업표준서 — Excel/Word4. 자주검사항목 — Excel\---## 핵심 기술 스택### 최선의 아키텍처```사용자 챗봇 인터페이스 (React)        ↓ Session APIClaude Managed Agents (Anthropic 관리형 인프라)        ↓Orchestrator Agent (총괄)  ├→ FMEA Agent          ┐  ├→ CP Agent            ├ 병렬 실행 (Parallel Workflow)  ├→ 작업표준서 Agent    │  └→ 자주검사 Agent      ┘        ↓ MCP 서버 연동  ├→ Vector DB (사내 기존 문서 500개+ 유사 사례 검색)  └→ LLM Wiki (AIAG/IATF/CSR 룰셋 자동 관리)        ↓Excel 출력 (4종 동시 생성)```### 기술 선택 근거* **Claude Managed Agents** : Agent 루프/인프라 직접 구축 불필요. Anthropic이 관리.* **Agentic RAG** : 단순 RAG 대비 공정 유형별 동적 판단 가능. Router → 전문 Agent 구조.* **LLM Wiki (Karpathy 패턴)** : AIAG/IATF/CSR 룰셋을 raw/ 폴더에 넣으면 LLM이 자동 정리.* **파인튜닝은 사용하지 않음** : 데이터 변경 시 재학습 비용 과다. 멀티테넌트 구조에 부적합.\---## 데이터 구조### 사내 학습 데이터* 기존 FMEA
