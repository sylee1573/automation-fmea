> ② 출력 품질 격차 해결, ③ 첫 파일럿/구매자 확보 — 세 가지가 갖춰진 뒤.

## 목표
구매자(비기술자 품질/공정 담당자) PC에서 **Python/Node 설치 없이, 완전 오프라인으로, 더블클릭 한 번**으로
구동되는 올인원 번들. 미지의 PC 환경(포트 충돌·권한·백신·폐쇄망·런타임 부재)에 대한 대비책 포함.

확정 전제(사용자 결정):
- 배포 = 임베디드 파이썬 번들 + .bat 런처 (완전 오프라인)
- RAG ML 스택(torch·sentence-transformers·chromadb) + 임베딩 모델까지 올인원 동봉
- Claude API 키는 구매자가 자기 키 입력(현행 `/setup/apikey` 유지)

## 핵심 제약
- **개발 머신 Python 3.14**는 torch/chromadb 휠(cp314) 부재 → **번들·휠은 CPython 3.12 win_amd64 고정**.
- 임베디드 zip은 pip 없음 + `pythonXX._pth`가 site-packages 비활성 + torch용 `vcruntime140_1.dll` 부재
  → **python-build-standalone(indygreg/astral) install_only 풀 인터프리터** 채택(폴더 복사만으로 동작).
- 설치 폴더에 output·data/chroma·설치마커 기록 → **Program Files 금지**(권장 `C:\표준류` 또는 `%LOCALAPPDATA%\표준류`).

## 번들 폴더 레이아웃
```
표준류_자동생성_시스템\
├─ 표준류_실행.bat              ← 더블클릭 진입점
├─ 시작하기_먼저_읽으세요.txt    ← 비기술자용 온보딩/문제해결 안내
├─ bootstrap.py                ← 첫 실행 점검/오프라인 설치/자가치유
├─ python\                     ← 번들 CPython 3.12 (pip 포함, site-packages 사전설치)
├─ app\
│   ├─ backend\                ← backend 패키지 전체 (wiki\wiki\*.md 포함)
│   ├─ frontend\dist\          ← npm run build 산출물
│   ├─ data\chroma\            ← Vector DB
│   ├─ output\                 ← 생성물 출력
│   └─ run.py                  ← 런처 (자유포트+브라우저+env+uvicorn)  ※ 씬 슬라이스로 이미 구현됨
├─ wheels\                     ← 오프라인 휠 캐시(torch CPU/chromadb/... 전 closure)
├─ models\paraphrase-multilingual-MiniLM-L12-v2\  ← 번들 HF 임베딩 모델
├─ redist\VC_redist.x64.exe    ← torch 런타임 의존
└─ logs\
```

## 작업 항목
1. **런처 run.py** — 자유 포트(8000→8001→8080→8765→bind(0)), 브라우저 자동 오픈, 단일 프로세스, host 127.0.0.1.
   번들에서는 오프라인 env(`HF_HUB_OFFLINE`, `TRANSFORMERS_OFFLINE`, `SENTENCE_TRANSFORMERS_HOME`,
   `STANDARDS_MODEL_DIR`) 설정. **※ 개발용 골격은 씬 슬라이스로 이미 구현·검증 완료.**
2. **embeddings.py get_model()** — `STANDARDS_MODEL_DIR` 로컬 경로 분기. **※ 이미 구현 완료.**
3. **표준류_실행.bat** — python 점검 → bootstrap → run.py, 한국어 오류/pause/로그 안내.
4. **bootstrap.py** — 설치마커+smoke import 점검, 깨졌으면 `pip install --no-index --find-links wheels`로 자가치유.
   환경 대비책: Windows 버전, `vcruntime140_1.dll`, 설치폴더 쓰기권한, `logs\startup.log`+`~/.standards_gen/last_run.log` 로깅.
5. **시작하기_먼저_읽으세요.txt** — 짧은 경로 권장, SmartScreen 통과법, API 키 입력 위치, 백신 예외, 연락처.
6. **build_bundle.ps1** (판매 머신, 인터넷 1회) — python-build-standalone 3.12 풀기 → 프론트 빌드 →
   app 복사 → **번들 3.12로** `pip download`(torch CPU 빌드 `download.pytorch.org/whl/cpu`) →
   HF 모델 `.save()` → site-packages 사전설치 → VC++ 동봉 → zip.

## 미해결 결정
- `chromadb → sqlite-vec` 교체 여부 — 교체 시 토치 번들(가장 무거운 부분) 전체가 단순화/제거됨.
- `data\chroma` 사전 색인본 동봉 vs 현장 색인(멀티테넌트 격리 주의).
- SmartScreen 근본 해결용 코드서명 인증서(별도 비용).
