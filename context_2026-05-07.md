## 6. 오늘 발견한 문제 3가지

### 🔴 문제 1: RAG가 생성 파이프라인과 연결이 끊겨 있음 (가장 중요)

`retriever.py`는 완성되어 있으나, `main.py`가 generate 호출 시 RAG를 사용하지 않음.

```python
# backend/main.py (현재 — 문제 있는 코드)
result = await run_sequential(
    ...
    similar_cases="",   # ← 항상 빈 문자열. retriever.search() 호출 없음
)
```

**필요한 작업**: `main.py`의 `/generate` 핸들러에서 retriever를 호출해 `similar_cases`를 채워야 함.

```python
# 수정 예시 (3~4줄)
from backend.rag.retriever import search, format_for_prompt

similar_results = search(
    query=f"{session['process_type']} {session['part_name']}",
    process_type=session.get("process_type", ""),
)
similar_cases = format_for_prompt(similar_results)
```

### 🟡 문제 2: 학습 데이터 자체가 없음 (담당자 작업)

기존 FMEA Excel 500개가 아직 인덱싱되지 않았음. Vector DB 비어있음.

| 단계 | 스크립트 | 실행 여부 |
|---|---|---|
| 파일 스캔 | `excel_scanner.py` | ❌ 미실행 |
| 메타 태그 | `metadata_tagger.py` | ❌ 미실행 |
| 품질 분류 | `quality_filter.py` | ❌ 미실행 |
| DB 인덱싱 | `indexer.py` | ❌ 미실행 |

실행 순서:
```bash
python backend/preprocessing/excel_scanner.py --dir <공유서버경로> --output 목록표.csv
python backend/preprocessing/metadata_tagger.py --input 목록표.csv
python backend/preprocessing/quality_filter.py --input 목록표_tagged.csv
python backend/rag/indexer.py --input 목록표_graded.csv --customer-id default
```

### 🟡 문제 3: 코드가 구현계획 결정사항과 불일치

| 항목 | 구현계획.md 결정 (PATCH-8) | 현재 코드 |

## 7. 실제 테스트에 필요한 것

### 필수
| 항목 | 내용 |
|---|---|
| Anthropic API 키 | `sk-ant-api03-...` 형식. 앱 실행 시 입력 화면 자동 표시 |
| 테스트 파일 | 도면 PDF 1개 또는 공정검토서 Excel 1개 (없으면 업로드 자체 불가) |

### 없어도 됨
- 라이선스 키 (현재 생성 차단 안 함)
- Vector DB 데이터 (없으면 "유사 사례 없음"으로 처리, 에러 없음)
- wiki/raw/ 원본 문서 (이미 wiki/*.md 4개 있음)

### 파일 없이 curl로 데모 테스트
```bash
# 1. 파일 없이 세션 생성
curl -X POST http://localhost:8000/upload
# → {"session_id":"xxx","summary":"파일 없음 — 데모 시나리오로 생성합니다"}

# 2. 생성 요청
curl -X POST http://localhost:8000/generate \
  -d "session_id=xxx&pfmea=true&cp=true&work_standard=true&inspection=true"
```
