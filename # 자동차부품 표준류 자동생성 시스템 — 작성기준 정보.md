## 1. PFMEA (공정 FMEA) — AIAG & VDA 1st Edition (2019)

### 1.1 7단계 접근법 (7-Step Approach)

| Step | 명칭 | 핵심 활동 |
|------|------|-----------|
| 1 | Planning & Preparation | 범위 정의, 프로젝트 계획, 팀 구성, Process Flow Diagram(PFD) 입력 |
| 2 | Structure Analysis | 제조시스템을 Process Item → Process Step → Process Work Element로 분해 |
| 3 | Function Analysis | 각 공정 단계와 작업 요소의 기능/요구사항 식별 |
| 4 | Failure Analysis | 고장 체인(Failure Chain) 수립: Failure Mode → Effect → Cause |
| 5 | Risk Analysis | S/O/D 평가 + AP(Action Priority) 결정 |
| 6 | Optimization | 리스크 저감 개선 조치 도출 및 책임자/일정 지정 |
| 7 | Results Documentation | 결과 문서화, 보고서 공유 |

### 1.3 AP(Action Priority) 판정 기준 — RPN 대체

| AP 등급 | 의미 | 조치 의무 | 적용 기준 |
|---------|------|-----------|-----------|
| H (High) | 최우선 | 반드시(MUST) 개선 | S=9~10 또는 고위험 S/O/D 조합 |
| M (Medium) | 중간 우선 | 개선 권고(SHOULD) | 중간 위험 S/O/D 조합 |
| L (Low) | 낮은 우선 | 개선 가능(COULD) | 현 수준 허용 가능 |

> 핵심 변경사항 (2019): RPN 임계값(예: 100 초과) 기준 완전 폐기 → AP 테이블로 대체.  
> S/O/D 조합으로 AP 자동 계산 가능 (5×5×5 룩업 테이블).

### 1.4 사전 준비 문서 (시스템 입력값)

- Process Flow Diagram (PFD) → Step 2 Structure Analysis 입력
- Characteristics Matrix → 특별특성 식별
- Past FMEA / 8D → 과거 고장 데이터 활용
- Engineering Drawing → 기능/요구사항 추출

## 2. Control Plan (관리계획서) — AIAG APQP 기준

### 2.1 관리계획서 수준 (Level)

| 수준 | 시점 | 내용 |
|------|------|------|
| Prototype | 시제품 단계 | 측정 치수, 재료/성능 시험 |
| Pre-Launch | 양산 전 단계 | 측정 치수, 재료/성능 시험 |
| Production | 양산 단계 | 특별특성, 공정관리, 측정방법 전체 포함 |

### 2.2 헤더 섹션 필드 (JSON 스키마)

```json
{
  "control_plan_level": "Prototype | Pre-Launch | Production",
  "control_plan_number": "",
  "part_number": "", "latest_change_level": "",
  "part_name_description": "",
  "supplier_plant": "", "supplier_code": "",
  "key_contact": "", "phone": "",
  "core_team": "",
  "supplier_approval_date": "",
  "customer_engineering_approval_date": "",
  "customer_quality_approval_date": "",

```

### 2.3 본문 행(Row) 필드 — 공정별 특성 관리 (JSON 스키마)

```json
{
  "part_process_number": "",

  "char_number": "",

  "sample_size": "",
  "sample_frequency": "",
  "control_method_prevention": "",
  "control_method_detection": "",
  "reaction_plan": ""

```

### 2.4 특별특성 분류 기호

| 기호 | 의미 |
|------|------|
| D | Design (설계 특성) |
| P | Process (공정 특성) |
| R | Regulatory (법규 관련) |
| OS | Operator Safety (작업자 안전) |

> OEM별로 추가 기호 사용 가능 (CC, SC, 역삼각형, 별표 등 고객사 요구사항 반영 필요)

### 2.5 Control Plan 입력 소스 매핑

| CP 필드 | 입력 소스 |
|---------|-----------|
| process_name, part_process_number | Process Flow Diagram |
| product_characteristic, spec_tolerance | Engineering Drawing |
| special_char_classification | DFMEA/PFMEA (AP=H/M 항목) |
| control_method_detection | PFMEA Detection Controls |
| reaction_plan | PFMEA Optimization 조치 |

## 3. 자주검사일지 (Operator Self-Check Sheet)

> IATF 16949에서 별도 양식을 직접 규정하지 않음.  
> 관리계획서(Control Plan)의 검출 관리 항목에서 자동 파생되는 문서.

### 3.1 헤더 필드 (JSON 스키마)

```json
{
  "document_number": "",
  "part_number": "", "part_name": "",
  "process_name": "", "operation_number": "",
  "machine_id": "",
  "effective_date": "", "revision": "",
  "approved_by": ""
}
```

### 3.2 검사 항목 필드 (Control Plan에서 자동 추출)

```json
{
  "inspection_items": [{
    "item_no": "",
    "inspection_item": "",
    "drawing_symbol": "",
    "specification": "",
    "measurement_tool": "",
    "sample_size": "",
    "inspection_frequency": "",
    "special_characteristic": "",
    "record_columns": [
      "time_or_lot", "measured_value", "judgment_ok_ng", "operator_sign"
    ]
  }],
  "footer": {
    "nonconformance_action": "",
    "inspector": "",
    "supervisor_review": ""
  }
}
```

### 3.3 자주검사일지 항목 자동 필터링 기준

Control Plan에서 아래 조건에 해당하는 행을 자동 추출:

1. `special_char_classification` 값이 존재하는 행 (D/P/R/OS)
2. `sample_frequency` 가 설정된 행 (초물검사, N개마다, 최종검사)
3. PFMEA에서 AP=H 또는 M으로 분류된 항목과 연계된 특성

## 4. 3문서 자동생성 연계 흐름 (아키텍처)
[입력]
공정흐름도(PFD) 업로드  → 공정 단계 파싱
도면(Drawing) 업로드   → 치수/특성/공차 파싱
↓
[STEP 1: PFMEA 자동생성]
Step2(Structure)  ← PFD 공정단계
Step3(Function)   ← 도면 요구사항
Step4(Failure)    ← LLM 추론
Step5(Risk)       ← AP 룩업테이블
Step6(Optimize)   ← 고위험 항목 조치
↓ 특별특성 + 예방/검출관리 추출
[STEP 2: 관리계획서 자동생성]
공정명        ← PFD
규격/공차     ← 도면
특별특성      ← PFMEA AP=H/M 항목
검출관리      ← PFMEA Detection Control
대응계획      ← PFMEA Optimization
↓ 검사항목 + 규격 + 측정기 추출
[STEP 3: 자주검사일지 자동생성]
검사항목  ← CP Product Characteristic
규격      ← CP Spec/Tolerance
측정기    ← CP Measurement Technique
주기      ← CP Sample Frequency
대응조치  ← CP Reaction Plan

## 6. Claude Code 구현 핵심 포인트

### 6.1 도면 파싱 시 인식 대상
- 치수 기호: 직경(원형), 사각, R(반경)
- 공차: ±, H/h (끼워맞춤), 기하공차 기호 (진직도, 평면도, 진원도 등)
- 특별특성 마크: CC, SC, 역삼각형, 별표 (고객사별 상이)

### 6.2 AP 자동 계산 로직
- S/O/D 값 입력 → AIAG VDA AP 룩업 테이블 (5×5×5 매트릭스) 참조
- PFMEA와 DFMEA의 AP 테이블이 약간 상이 → 분리 관리 필요

### 6.3 3문서 연계 자동화 핵심 로직
- PFMEA AP=H/M 항목 → Control Plan 특별특성으로 자동 마킹
- Control Plan `special_char_classification` 존재 행 → 자주검사일지 항목으로 자동 추출
- Control Plan `reaction_plan` → 자주검사일지 `nonconformance_action`으로 연동

### 6.4 OEM별 고객 특수 요구사항 (CSR) 고려
- 현대/기아, GM, Ford, BMW 등 OEM마다 특별특성 기호, AP 임계값, 문서 양식 상이
- 시스템에 OEM 선택 옵션 추가 권장
