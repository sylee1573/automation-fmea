# 자동차 부품 표준류 자동생성 시스템 — 구현 계획

## Context

자동차 부품 제조사(1~3차 협력사) 대상 판매용 Windows 소프트웨어.
도면(PDF)과 공정검토서(Excel)를 입력받아 FMEA·CP·작업표준서·자주검사항목 4종을 AI로 자동 생성.
Claude Code 스킬 중 `/harness`(에이전트 정의)와 `/gstack`(QA 테스트)을 개발 워크플로에 통합.

### 2026-06-29 — RAG 스택 정합 (계획서 ↔ 코드)

계획서에 남아있던 **sqlite-vec / OpenAI 임베딩** 표기가 실제 코드(**Chroma + 로컬 sentence-transformers MiniLM**)와 어긋나 있던 문제를 **계획서를 코드에 맞추는 방향으로** 정리.
- **결정 근거**: 자동차 부품사는 도면·FMEA가 핵심 IP라 OpenAI 임베딩(외부 API 전송)은 영업·보안상 불가 + 색인/쿼리마다 토큰 과금 영구 발생 + 오프라인 번들 배포 목표와 정면 충돌. 현행 로컬 임베딩은 무료·오프라인·사내잔류로 비용·고객편의 모두 우월.
- **벡터스토어 sqlite-vec 전환**은 검색 품질과 무관한 *패키징 경량화* 이슈로 격하 — 실제 번들이 Chroma 의존성(torch 등)으로 무거울 때만 검토. 미리 하지 않음.
- **한국어 검색 품질이 부족하다고 측정되면** OpenAI가 아니라 **로컬 한국어 특화 모델(ko-sroberta, bge-m3 등)** 로 교체하는 길로 해결.
- 본 정합으로 아래 §아키텍처·폴더구조·Phase 3·핵심 의존성의 sqlite-vec/OpenAI 표기를 현행 스택으로 수정. (코드 변경 없음, 문서만)

**End-to-End 검증 (실제 LLM 생성, API 키 입력 후)**
- 전체 파이프라인 완주: 업로드(데모)→RAG(found=True)→FMEA(19행)→CP(46행)→작업표준서(35)+자주검사(28, 병렬)→정합성→Excel 4종 생성·다운로드.
- RAG 실증: 출력 내용이 MTK 색인 사례 반영. 양식 실증: FMEA가 MTK `.xlsm` 양식 그대로 출력.
- **버그 수정**: 4개 생성 에이전트가 `max_tokens`(4096~8192) 초과로 JSON 잘림 → **streaming(`client.messages.stream`)+`max_tokens=64000`으로 전환**(Haiku4.5/Sonnet4.6 모두 64K 지원). orchestrator 압축기 2048→8192.
- 소요 ~6분(데모 시나리오가 행 다수 생성). 90초 목표는 실입력·행수 따라 별도 튜닝 필요.

**순차 실행 패턴 (MVP) — GenerationOptions 포함**
```python
from dataclasses import dataclass

@dataclass
class GenerationOptions:
    pfmea: bool = True
    cp: bool = True
    work_standard: bool = True
    inspection: bool = True

    def validate(self):
        """의존성 자동 보정 (CP는 FMEA 없이 생성 불가 등)"""
        if self.cp and not self.pfmea: self.pfmea = True
        if self.work_standard and not self.cp: self.cp = True; self.pfmea = True
        if self.inspection and not self.cp: self.cp = True; self.pfmea = True

async def run_sequential(process_data, options: GenerationOptions):
    options.validate()
    fmea_result = await fmea_agent.run(process_data) if options.pfmea else None
    if options.cp and fmea_result:
        fmea_summary = await compress_for_next_agent(fmea_result, "CP")  # Haiku 압축기
        cp_result = await cp_agent.run(process_data, fmea_summary)
    if options.work_standard and cp_result:
        cp_summary = await compress_for_next_agent(cp_result, "작업표준서")
        ws_result = await work_standard_agent.run(process_data, cp_summary)
    if options.inspection and cp_result:
        cp_summary = await compress_for_next_agent(cp_result, "자주검사")
        insp_result = await inspection_agent.run(process_data, cp_summary)
    # 속도 불만이 실제 고객 피드백으로 확인되면 이후 병렬화
```

## 하드웨어 지문 라이선스 (`backend/license.py`)

로컬 설치 구조에서 무단 복사·사내 무제한 배포를 통제하기 위해 도입.

- CPU ID + MAC 주소 조합 SHA256 해시로 기기 고유값 생성
- 납품 시 해당 PC 전용 라이선스 키 발급, 타 PC 사용 차단
- 만료 30일 전 갱신 안내 배너, 만료 후 실행 차단
- 인터넷 없이 로컬에서 검증 (오프라인 환경 지원)
- PC 교체·고장 시 재발급: 개발사 이메일로 기존 지문값 + 새 지문값 제출 → 영업일 1일 이내 신규 키 발급
  - ⚠️ [TODO-LICENSE-PORTAL] 고객사 10개 초과 시 자동 재발급 포털 구축 검토

```python
def get_hardware_fingerprint() -> str:
    mac = ':'.join(['{:02x}'.format((uuid.getnode() >> e) & 0xff)
                    for e in range(0, 2*6, 2)][::-1])
    result = subprocess.run(
        ['powershell', '-Command',
         'Get-CimInstance Win32_Processor | Select-Object -ExpandProperty ProcessorId'],
        capture_output=True, text=True)
    cpu_id = result.stdout.strip()
    return hashlib.sha256(f"{mac}:{cpu_id}".encode()).hexdigest()[:32]
```
